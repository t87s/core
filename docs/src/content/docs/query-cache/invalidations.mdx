---
title: Invalidations
description: Tell the cache what just changed
---

import { Tabs, TabItem } from '@astrojs/starlight/components';

Invalidations are the promise you keep. You changed the data, so you say so, and everything depending on it stops pretending it's fresh.

<Tabs>
  <TabItem label="TypeScript">
    ```typescript
    await cache.invalidate(cache.tags.users('123'));
    await cache.invalidate(cache.tags.users);
    await cache.invalidate(cache.tags.users('123'), true);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    await cache.invalidate(cache.t.users("123"))
    await cache.invalidate(cache.t.users)
    ```
  </TabItem>
</Tabs>

The first line invalidates one user. The second invalidates all users. The third is exact invalidation, which TypeScript supports directly or via primitives. Python will catch up.
